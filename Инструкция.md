# GryFf — Полная спецификация проекта

## 1. Введение и общая концепция
**GryFf** — это кроссплатформенный мессенджер, объединяющий лучший опыт Telegram, ВКонтакте и Discord.  
- **Ядро**: архитектура и базовый интерфейс Telegram (списки чатов, облака сообщений, каналы).  
- **Сообщества**: полноценные паблики и группы как в ВК (стена, обсуждения, медиа, мероприятия).  
- **Звонки**: голосовые и видеозвонки, голосовые каналы, стриминг экрана как в Discord.  
- **Профиль пользователя**: личная страница с фотографиями, записями на стене, друзьями (как в ВК).  
- **Дизайн**: база — Telegram, но с более плавными анимациями, серым фоном #808080, оранжевым акцентом #FF8C00 (минимально, только для важных элементов).

## 2. Функциональные требования (детально)

### 2.1. Аутентификация и пользователи
- **Регистрация** по номеру телефона (SMS-код), как в Telegram.  
- **Двухфакторная аутентификация** (пароль).  
- **Восстановление доступа** через email (привязка почты опционально).  
- **Сессии**: возможность видеть активные сессии и завершать их удалённо.  
- **Контакты**: автоматическая синхронизация с телефонной книгой (с согласия), поиск по номеру/имени.  

### 2.2. Чаты и сообщения (Telegram-подобные)
- **Типы чатов**:  
  - Личные (один на один).  
  - Групповые (до 200 000 участников).  
  - Каналы (неограниченная аудитория).  
- **Сообщения**:  
  - Текст, форматирование (жирный, курсив, моноширинный, ссылки).  
  - Медиа: фото (сжатие/оригинал), видео, аудио, документы, голосовые сообщения.  
  - Стикеры и GIF: встроенные наборы, возможность добавлять свои.  
  - Реакции (эмодзи) на сообщения.  
  - Ответы, пересылка, редактирование (история правок), удаление для всех.  
  - Закреплённые сообщения в чате.  
- **Уведомления**: настраиваемые для каждого чата (звук, вибрация, отключение).  
- **Поиск**: по тексту, дате, отправителю, медиа.  
- **Архив чатов**: скрытие чатов из основного списка.  

### 2.3. Личный профиль (как в ВК)
- **Страница пользователя**:  
  - Аватарка (круглая), имя, фамилия, статус (онлайн/офлайн).  
  - **Стена профиля**: записи (текст, фото, видео), комментарии, лайки, репосты.  
  - **Друзья**: список друзей с возможностью добавления/удаления.  
  - **Фотографии**: альбомы с фотографиями (личные, загруженные).  
  - **Информация**: дата рождения, город, о себе, контакты.  
- **Настройки приватности**: кто может видеть стену, список друзей, фотографии (все, только друзья, только я).  
- **Лента новостей** (опционально): показывает записи друзей и сообществ, на которые подписан пользователь.  

### 2.4. Сообщества (как в ВК)
- **Типы сообществ**:  
  - Публичная страница (для брендов, СМИ) — открытая подписка, стена, обсуждения.  
  - Группа (по интересам) — может быть открытой/закрытой, с возможностью вступления.  
  - Мероприятие (с датой, местом, участниками).  
- **Структура сообщества**:  
  - **Шапка**: обложка, аватар, название, тип, количество подписчиков, кнопка действия (подписаться/вступить).  
  - **Вкладки**:  
    - **Стена**: посты (текст, медиа, опросы, закреплённый пост), лайки, комментарии, репосты.  
    - **Обсуждения**: форум с темами и ответами (как в группах ВК).  
    - **Медиа**: альбомы с фото/видео.  
    - **Участники**: список с ролями (админ, модератор, редактор, участник), поиск.  
    - **Статистика**: охват, активность, демография.  
- **Управление**:  
  - Создание поста от имени сообщества.  
  - Назначение ролей.  
  - Черный список.  
  - Настройки приватности (открытое/закрытое/частное).  
- **Мероприятия**:  
  - Указание времени, места, возможность отметить участие.  
  - Обсуждения мероприятия.  

### 2.5. Звонки и голосовые каналы (как в Discord)
- **Индивидуальные звонки**: аудио/видео (P2P + relay при необходимости).  
- **Групповые звонки**: до 25 человек (SFU архитектура).  
- **Голосовые каналы в сообществах**:  
  - Постоянные комнаты, куда участники могут заходить и выходить.  
  - Отдельные голосовые каналы с названиями, ограничением по участникам.  
  - Видео и стриминг экрана внутри канала (может вещать один или несколько).  
- **Управление голосовыми каналами**:  
  - Создание, удаление, переименование (только админы).  
  - Права доступа: например, только определённая роль может говорить.  
  - Push-to-talk или голосовая активность.  
- **Качество**:  
  - Адаптивный битрейт, Opus кодек.  
  - Шумоподавление (RNNoise или AI-based).  
  - Эхоподавление, автоматическая регулировка усиления.  
- **Запись звонков** (опционально, с согласия всех участников).  

### 2.6. Дополнительные функции (изюминки)
- **Плавные анимации**: переходы между экранами (shared element transition для аватарок, медиа).  
- **Кастомизация**:  
  - Светлая/тёмная тема (автоматически по системе).  
  - Выбор акцентного цвета (по умолчанию #FF8C00).  
- **Умные уведомления**: группировка по чатам, возможность ответить из уведомления.  
- **Встроенный переводчик** сообщений (Google Translate API / Yandex).  
- **Режим «невидимки»**: скрыть онлайн-статус для всех или для отдельных контактов.  
- **Секретные чаты**: end-to-end шифрование (протокол Signal), запрет на пересылку, самоуничтожение сообщений.  
- **Боты**: API для создания ботов (как в Telegram), поддержка инлайн-режима, клавиатур.  
- **Интеграции**: встраивание видео с YouTube, музыки со Spotify, подкастов.

## 3. Дизайн и UI/UX (детально)

### 3.1. Цветовая палитра
- **Основной фон**: #808080 (серый) во всех вариациях (основной экран, чаты, профиль).  
  - Допустимы оттенки: #F5F5F5 (очень светлый) для карточек, #E0E0E0 для разделителей.  
- **Акцентный цвет**: #FF8C00 (оранжевый) используется строго:  
  - Логотип, иконка приложения.  
  - Кнопка отправки сообщения (в поле ввода).  
  - Индикатор непрочитанных сообщений.  
  - Выделение активной вкладки.  
  - Кнопка звонка, создания чата/сообщества.  
- **Текст**:  
  - Основной: #000000 (на светлом фоне), #FFFFFF (на тёмном).  
  - Вторичный: #757575.  
  - Ссылки: акцентный цвет.  
- **Фон сообщений**:  
  - Свои сообщения: #FF8C00 с белым текстом (или обводка).  
  - Чужие: #F0F0F0 (светло-серый) с чёрным текстом.  
- **Тени**: лёгкие, размытие 4-8px, непрозрачность 10-15%.  

### 3.2. Типографика
- **Основной шрифт**: SF Pro (для iOS/macOS), Roboto (для Android/Web) — sans-serif.  
- **Размеры**:  
  - H1 (заголовок экрана): 24px, bold.  
  - H2 (название чата): 20px, semibold.  
  - Основной текст: 16px, regular.  
  - Второстепенный: 14px, regular, цвет #757575.  
  - Время и счётчики: 12px, regular.  

### 3.3. Иконки
- **Стиль**: линейные, толщина линии 2px, скруглённые концы.  
- **Размер**: 24x24px для панели навигации, 20x20px для действий внутри экрана.  
- **Анимация**: при нажатии иконка слегка увеличивается (scale 1.1) и возвращается.  

### 3.4. Экран «Список чатов» (как Telegram)
- **Верхняя панель**:  
  - Слева: меню (бургер) или аватар профиля (переход в профиль).  
  - По центру: название «GryFf» или «Чаты».  
  - Справа: иконка поиска (лупа), иконка создания нового чата (карандаш/плюс).  
- **Список чатов**:  
  - Каждый элемент: аватарка (круглая 50x50), имя чата, последнее сообщение (обрезанное), время, счётчик непрочитанных (оранжевый кружок).  
  - При долгом нажатии — контекстное меню: закрепить, прочитать всё, удалить, архив.  
  - Свайп влево: закрепить/открепить, свайп вправо: прочитать всё.  
- **Поиск**: при вводе фильтрация списка, подсветка совпадений.  

### 3.5. Экран «Чат»
- **Верхняя панель**:  
  - Назад (стрелка), аватарка и имя собеседника/группы (при нажатии — информация о чате).  
  - Справа: иконка звонка (аудио/видео), иконка меню (поиск, очистить историю и т.д.).  
- **Область сообщений**:  
  - Сообщения группируются по дням, с датой-разделителем.  
  - Свои сообщения справа (оранжевый фон или обводка), чужие — слева (серый фон).  
  - Аватарка собеседника показывается только у первого сообщения в группе подряд.  
  - Под сообщением: время, статус (отправлено/доставлено/прочитано).  
  - Закреплённое сообщение — показывается над областью сообщений (можно скрыть).  
  - При скролле вверх — подгрузка более старых сообщений.  
- **Панель ввода**:  
  - Слева: иконка стикеров/GIF, иконка прикрепления (галерея, файл, голосовое).  
  - Поле ввода (multiline, растёт до 5 строк).  
  - Справа: иконка отправки (оранжевая) или голосового сообщения (микрофон).  
  - Для голосовых: запись по удержанию, свайп вверх для блокировки.  
- **Контекстное меню сообщения** (долгое нажатие): ответить, скопировать, переслать, редактировать (своё), удалить, закрепить, выбрать.  

### 3.6. Экран «Профиль пользователя» (как в ВК)
- **Шапка**:  
  - Обложка (можно установить), аватарка (крупная, 100x100), имя и фамилия, статус.  
  - Кнопки: «Добавить в друзья» (если не друг), «Написать сообщение», «Позвонить».  
  - Для своего профиля: кнопка «Редактировать профиль».  
- **Вкладки под шапкой**:  
  - **Стена**: лента записей пользователя (каждая запись: текст, медиа, дата, лайки, комментарии, репосты). Возможность создать запись (только для своего профиля).  
  - **Фотографии**: сетка фото (3 колонки), при нажатии открывается галерея.  
  - **Друзья**: список друзей (аватарка, имя), общее количество.  
  - **Информация**: дата рождения, город, о себе, контакты.  
- **Навигация**: кнопка «Назад» в левом верхнем углу.  

### 3.7. Экран «Сообщество»
- **Шапка**: обложка (широкая), аватар, название, тип, количество подписчиков.  
  - Кнопка действия: «Подписаться» / «Вступить» / «Вы подписаны».  
  - Для администратора: кнопка «Управление».  
- **Вкладки**:  
  - **Стена**: аналогично профилю, но посты от имени сообщества.  
  - **Обсуждения**: список тем (название, автор, количество ответов, последнее сообщение).  
  - **Медиа**: альбомы (название, обложка, количество фото).  
  - **Участники**: список с ролями, поиск.  
  - **О сообществе**: описание, ссылки, дата создания.  
  - **Статистика** (для админов): графики активности.  
- **Дополнительно**:  
  - Возможность прикрепить голосовой канал к сообществу — тогда появляется вкладка «Голосовые каналы» со списком комнат.  

### 3.8. Экран «Звонок»
- **Входящий звонок**:  
  - Аватарка, имя, тип звонка (аудио/видео).  
  - Кнопки: принять (зелёная) / отклонить (красная).  
- **Активный звонок**:  
  - Для 1-на-1: видео крупно (собеседник), маленькое окно себя.  
  - Для группы: сетка участников (меняется при стриминге экрана).  
  - Снизу панель управления: микрофон, видео, стриминг экрана, завершить, динамик.  
  - При стриминге экрана: основной экран стримера, остальные — маленькие окна.  
- **Голосовые каналы**:  
  - Экран со списком каналов слева (как в Discord), справа — участники выбранного канала.  
  - При входе в канал открывается мини-панель с управлением, сам экран может оставаться в фоне (чат/сообщество).  

### 3.9. Анимации и переходы
- **Переход между экранами**: iOS-style push (справа налево) с затуханием.  
- **Открытие чата**: общая аватарка перемещается на место в шапке (shared element).  
- **Отправка сообщения**: плавное появление облака, скролл вниз.  
- **Нажатие на иконки**: ripple-эффект оранжевым цветом.  
- **Потянуть для обновления**: кастомный индикатор загрузки (оранжевая спираль).

## 4. Технические требования

### 4.1. Платформы и языки
- **iOS**: Swift, UIKit + SwiftUI (постепенно). Минимальная версия iOS 14.  
- **Android**: Kotlin, Jetpack Compose. Минимальная API 24.  
- **Desktop**: Electron (для быстрого старта) или нативные (C++/Qt) позже.  
- **Web**: React + TypeScript, PWA.  

### 4.2. Бэкенд
- **Язык**: Go (высокая производительность, конкурентность).  
- **Базы данных**:  
  - PostgreSQL (основные данные: пользователи, чаты, сообщения, посты).  
  - Redis (кэш, сессии, очереди уведомлений).  
  - Cassandra или ScyllaDB для огромных объёмов сообщений (опционально).  
- **Хранилище медиа**: S3-совместимое (MinIO, AWS S3).  
- **Протоколы**:  
  - WebSocket для real-time событий.  
  - REST API для запросов.  
  - WebRTC для звонков.  
- **Шифрование**:  
  - Все соединения через TLS.  
  - Секретные чаты: Double Ratchet (Signal Protocol).  
- **Микросервисы**:  
  - API Gateway (Nginx/OpenResty).  
  - Auth service.  
  - Chat service.  
  - Media service.  
  - Call service (SFU).  
  - Notification service.  

### 4.3. Звонки (WebRTC SFU)
- **SFU**: Mediasoup или Janus.  
- **TURN/STUN серверы**: Coturn.  
- **Кодеки**: VP8/VP9 для видео, Opus для аудио.  
- **Управление комнатами**: через сигнальный сервер (WebSocket).  

### 4.4. Безопасность
- **2FA**: TOTP (Google Authenticator) или SMS.  
- **Защита от брутфорса**: rate limiting, CAPTCHA.  
- **Антивирус для загружаемых файлов** (ClamAV).  
- **Аудит логов**.

## 5. API спецификация (основные эндпоинты)

### 5.1. Аутентификация
- `POST /auth/sendCode` — отправка кода на номер.  
- `POST /auth/verifyCode` — проверка кода, получение токена.  
- `POST /auth/logout` — выход.  

### 5.2. Чаты
- `GET /chats` — список чатов.  
- `GET /chats/{id}/messages` — история сообщений.  
- `POST /chats/{id}/messages` — отправить сообщение.  
- `PUT /messages/{id}` — редактировать.  
- `DELETE /messages/{id}` — удалить.  
- `POST /chats` — создать чат.  

### 5.3. Сообщества
- `GET /communities` — список сообществ пользователя.  
- `GET /communities/{id}` — информация.  
- `POST /communities` — создать сообщество.  
- `POST /communities/{id}/posts` — создать пост на стене.  
- `GET /communities/{id}/posts` — лента стены.  

### 5.4. Звонки
- `POST /calls/start` — инициировать звонок (возвращает сигнальные данные).  
- `POST /calls/{id}/join` — присоединиться.  
- `POST /calls/{id}/signal` — обмен SDP.  

*(Полная спецификация в OpenAPI должна быть разработана отдельно.)*

## 7. Дизайн-система (компоненты)
Для единообразия необходимо создать библиотеку компонентов в коде:  
- **Avatar**: круглая, размеры S(32), M(40), L(50), XL(80).  
- **Button**: Primary (оранжевый фон), Secondary (серая обводка), Icon (только иконка).  
- **Input**: поле ввода с placeholder, иконкой слева/справа.  
- **MessageBubble**: два типа (outgoing, incoming) с поддержкой медиа.  
- **TabBar**: переключение вкладок с индикатором.  
- **Modal**: для предпросмотра медиа, контекстных меню.  
- **Toast**: всплывающие уведомления (оранжевые).
