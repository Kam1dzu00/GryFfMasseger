### 1. КОНЦЕПЦИЯ И ФИЛОСОФИЯ ПРОЕКТА

**GryFf** — это гибридная платформа общения, объединяющая лучшее из трёх миров:

| Источник        | Что берём                                                                 |
|-----------------|---------------------------------------------------------------------------|
| Telegram        | Скорость, надёжность, простота интерфейса, базовая архитектура чатов     |
| ВКонтакте       | Глубина социальных взаимодействий: профили, стена, друзья, сообщества    |
| Discord         | Голосовые каналы, стриминг, гибкая система прав, качественные звонки     |

**Ключевая идея**: Мы не копируем существующие приложения, а синтезируем лучший опыт, добавляя уникальный минималистичный дизайн с акцентным цветом `#FF8C00` на спокойном сером фоне `#808080`.

**Принципы разработки**:
- Микросервисная архитектура с первого дня.
- Асинхронное взаимодействие через событийную шину.
- Проектирование под 100M+ пользователей.
- Наблюдаемость (метрики, логи, трейсы) в каждом сервисе.
- Безопасность на всех уровнях.
- Mobile First – все экраны разрабатываются сначала для мобильных устройств, затем адаптируются.
- Accessibility – поддержка VoiceOver, TalkBack, динамического размера шрифта.

**Проблемы, требующие решения (из TODO):**

**Вывод**: Проект находится на финальной стадии – ядро, социальные функции и звонки реализованы на современной масштабируемой архитектуре. Основные усилия должны быть направлены на завершение интеграции фронтенда, внедрение AI-функций и полировку продукта до состояния production-ready.

### 3. ЦЕЛЕВАЯ АРХИТЕКТУРА ПРОДУКТА

#### 3.1. Функциональные модули

1. **Аутентификация и аккаунт**
   - Регистрация по номеру телефона (SMS-код через Twilio).
   - 2FA (TOTP).
   - Управление сессиями (просмотр активных, удалённое завершение).
   - Приватность (онлайн-статус, блокировки, скрытие "печатает").

2. **Чаты и сообщения**
   - Личные, групповые (до 200k), каналы, секретные (E2E).
   - Форматирование текста (Markdown/HTML: bold, italic, underline, spoiler, monospace).
   - Медиа: фото (сжатие/оригинал), видео, аудио, документы, голосовые сообщения.
   - Стикеры (анимированные/статические), GIF (интеграция с Tenor/Giphy).
   - Реакции (эмодзи, настраиваемые).
   - Ответы, пересылка, редактирование (с историей правок), удаление для всех.
   - Закреплённые сообщения.
   - Умные уведомления (настройка звука, группировка, "тихие" чаты).

3. **Профиль пользователя (VK-style)**
   - Стена с постами, лайками, комментариями, репостами.
   - Друзья (добавление, удаление, заявки).
   - Фотоальбомы (автоматические и созданные пользователем).
   - Информация о себе (дата рождения, город, контакты).
   - Настройки приватности (кто видит стену, друзей, фото).

4. **Сообщества**
   - Типы: публичная страница, группа (открытая/закрытая/частная), мероприятие (с датой, местом).
   - Стена сообщества (посты, опросы, закреп).
   - Обсуждения (форум с темами и ответами).
   - Медиатека (альбомы).
   - Участники с ролями (админ, модератор, редактор).
   - Голосовые каналы (интеграция с Discord-модулем).
   - Статистика (охват, активность).

5. **Звонки и голосовые каналы (Discord-level)**
   - Индивидуальные аудио/видео (P2P + TURN).
   - Групповые звонки (до 25 чел, SFU mediasoup).
   - Постоянные голосовые каналы в сообществах.
   - Стриминг экрана.
   - Шумоподавление (AI-based RNNoise).
   - Push-to-talk / Voice Activity.
   - Управление правами (кто может говорить).

6. **Боты и AI-фичи**
   - Telegram-подобные боты (API, инлайн-режим, клавиатуры).
   - Умные ответы (NLU).
   - Автоперевод сообщений.
   - Спам-фильтр (ML).
   - Рекомендации контента (посты, сообщества, друзья).

#### 3.3. Микросервисная архитектура
[Client] <-- бинарный протокол/WebSocket/HTTP --> [API Gateway]
|
+----------+----------+----------------+----------------+----------+
| | | | | |
[Auth] [User] [Chat] [Message] [Community] [Media] ...
| | | | | |
[PostgreSQL] [PostgreSQL] [PostgreSQL] [ScyllaDB] [PostgreSQL] [MinIO]
| | | | | |
+----------+----------+----------------+----------------+----------+
|
[NATS/Kafka]
|
[Notification] [Search] [Call] ...

text

- **API Gateway** — единая точка входа, маршрутизация, rate limiting, SSL termination.
- **Auth Service** — регистрация, логин, JWT, 2FA, управление сессиями.
- **User Service** — профили, друзья, настройки приватности.
- **Chat Service** — управление чатами, участниками, правами.
- **Message Service** — отправка/получение сообщений, история, поиск (бинарный протокол).
- **Community Service** — сообщества, стена, обсуждения, роли.
- **Media Service** — загрузка, обработка, хранение медиа.
- **Call Service** — сигналинг WebRTC, управление комнатами, интеграция с mediasoup.
- **Notification Service** — push-уведомления (APNS, FCM), email.
- **Search Service** — индексация и поиск по сообщениям, постам, пользователям.
- **Bot Service** — API для ботов, вебхуки.

Все сервисы общаются через **NATS** (или Kafka) для событий.

#### 3.4. Протоколы и API

- **Бинарный протокол поверх TCP/WebSocket** — основной для реального времени (сообщения, статусы, звонки). Спецификация: сообщения сериализуются в бинарный формат с заголовком (длина, тип, флаги). Поддерживается сжатие, шифрование (AES-256), подтверждение доставки (ACK), восстановление сессии.
- **REST/HTTP** — для тяжёлых операций (регистрация, загрузка файлов).
- **gRPC** — для внутреннего взаимодействия микросервисов (высокая производительность).

#### 3.5. Базы данных и хранение

- **PostgreSQL**: пользователи, чаты (метаданные), сообщества, посты, друзья. Шардинг по user_id (партиционирование).
- **ScyllaDB/Cassandra**: история сообщений. Модель данных: ключ = (chat_id, time_uuid), значение = сообщение.
- **Redis**: кэш (профили, сессии), очереди уведомлений, Pub/Sub.
- **OpenSearch**: полнотекстовый поиск по сообщениям и контенту.
- **MinIO / S3**: медиафайлы. Имена объектов: `/{user_id}/{type}/{uuid}`.

#### 3.6. Инфраструктура и DevOps

- **Kubernetes** — оркестрация микросервисов (желательно Managed K8s, например, GKE, EKS).
- **Helm** — управление чартами.
- **Terraform** — управление инфраструктурой как кодом (облачные ресурсы).
- **GitHub Actions** — CI (тесты, сборка, публикация образов).
- **ArgoCD** — CD (синхронизация с Git).
- **Prometheus + Grafana** — метрики и дашборды.
- **Loki** — агрегация логов.
- **Jaeger** — трассировка запросов.

#### 3.7. Безопасность

- **mTLS** между микросервисами.
- **Шифрование данных в покое** (БД, S3).
- **JWT с коротким сроком жизни** + refresh tokens.
- **Rate limiting** на API Gateway.
- **Защита от брутфорса** (fail2ban, капча).
- **Антивирус** для загружаемых файлов (ClamAV).
- **Double Ratchet** для секретных чатов (по протоколу Signal).

#### 5.5. Анимации и переходы

- **Shared Element Transition**: Аватарка из списка чатов плавно перемещается в шапку экрана чата.
- **Push/Pop**: iOS-style переход справа налево с лёгким затуханием фона.
- **Сообщения**: появляются с анимацией масштабирования (0.9 → 1) и смещения, список плавно скроллится вниз.
- **Отправка**: иконка отправки превращается в спиннер, затем в галочку.
- **Нажатия**: ripple-эффект оранжевого цвета на кнопках и кликабельных элементах.
- **Обновление списка**: кастомный индикатор загрузки (вращающееся оранжевое кольцо с прогрессом).
- **Переключение вкладок**: плавное подчёркивание с анимацией ширины.
- **Голосовые сообщения**: волна амплитуды при записи.
- **Звонок**: при входящем звонке — пульсация аватарки.
